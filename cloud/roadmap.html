--- title: Roadmap ---
Roadmap
2019 Q2
Faster rank function without term position
Using term position information normally improves relevance. Not all applications need this / requires higher query throughput
Export metrics to monitoring system
Vespa processes on nodes expose a JSON metrics interface. These metrics can be scraped and integrated with monitoring systems. With the new metrics proxy, all metrics on a node are aggregated in one interface, and renamed as needed for easier integration
Multiphased grouping execution in backend
Currently Vespa executes the full grouping in the content node in a single phase if that is necessary. This is necessary if at any level there is ordering on something else than rank. In the case of multiple levels with different ordering, this is not optimal. Instead do multi pass in the content node
System Test framework
Enable external contributors to validate code at system level
TLS on internal protocols
All intra-process communication is secured using TLS
Support continuously or periodically trained machine learned models in ranking
Enables pulling models (TensorFlow/ONNX/XGBoost) from external storage
2019 Q1
Release Vespa 7
First major version bump of Vespa in >2 years that allows us to break backwards compatibility. Necessary to clean up deprecated APIs/code, migrate to Java 11 and prepare for the new, faster Java release cycle. This modernizes code and improves security for Vespa applications. Release notes
Vespa advanced ranking models: Partial update of document tensor cells
Support online and reinforcement learning in models with large parameter spaces, and continuous updates of individual signals stored in tensors. API documentation
Boolean document field type
Support modelling data more precisely to save memory and memory bandwidth. Read more on selecting field type
